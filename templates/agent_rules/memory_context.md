# Memory And Context

## 原则

- 长期上下文只保存稳定、必要、可复用的信息。
- 近期对话、原始日志和临时推理不直接写入长期上下文。
- 不写入非必要个人隐私、客户隐私、密钥、会话 ID 或未经确认的事实。

## 建议文件

- `long_term_context.md`：长期稳定信息。
- `chat_history.md`：近期原始对话或摘要。
- `context_review.md`：待审核的记忆候选。

这些文件属于真实运行数据，应保存在私有工作区，不提交到框架仓库。

## 压缩时保留

- 用户已确认的决定。
- 当前环境约束。
- 已修改文件和原因。
- 已运行命令和关键结果。
- 未完成事项、阻塞点和下一步。
- 用户明确说过不要做的事项。
