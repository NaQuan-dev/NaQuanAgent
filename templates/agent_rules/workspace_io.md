# Workspace IO

## 文件原则

- 修改文件前先读取现有内容。
- 不覆盖用户已有改动，除非用户明确要求。
- 临时文件放入 `<TEMP_DIR>`。
- 日志放入 `<LOG_DIR>`。
- 正式产物放入用户指定目录或对应业务目录。

## 编码

- 中文 Markdown、JSON、CSV、日志和脚本优先使用 UTF-8。
- 需要传递长文本、中文 JSON 或多行内容时，优先使用 UTF-8 文件、标准输入或 Base64。
- 写入后抽查关键中文片段，不能只看退出码。

## 清理

- 清理临时目录前先预览范围、数量、大小和保留项。
- 未得到明确确认前不删除。
