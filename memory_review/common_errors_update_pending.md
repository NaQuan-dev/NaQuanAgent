# COMMON_ERRORS Update Candidates

Use this file during weekly memory review to collect candidate entries for root `COMMON_ERRORS.md`. Candidates store generic engineering and runtime lessons only. Do not store real organization names, employees, customers, sessions, secrets, private path details, full logs, or business-system private information.

## Rules

- Automations may add or organize candidates, but they must not merge directly into `COMMON_ERRORS.md`.
- A candidate must be reusable as a generic issue. One-off incidents tied only to a specific business fact do not belong here.
- Each candidate must include: scenario, typical symptoms/errors, likely cause, preflight strategy, and verification.
- If an existing entry already covers the issue, write a proposed supplement or rewrite instead of adding a duplicate.
- Merge into `COMMON_ERRORS.md` only after administrator review.
